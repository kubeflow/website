    <article class="markdown-body entry-content p-3 p-md-6" itemprop="text"><p>+++
title = "IBM Cloud Private for Kubeflow"
description = "Get Kubeflow running on IBM Cloud Private"
weight = 2
+++</p>
<p>This guide is a quick start to deploy Kubeflow on <a href="https://www.ibm.com/cloud/private" rel="nofollow">IBM Cloud Private</a> 3.1.0 or later.  IBM Cloud Private is an enterprise PaaS layer for developing and managing on-premises, containerized applications. It is an integrated environment for managing containers that includes the container orchestrator Kubernetes, a private image registry, a management console, and monitoring frameworks.</p>
<h3><a id="user-content-prerequisites" class="anchor" aria-hidden="true" href="#prerequisites"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Prerequisites</h3>
<ul>
<li>
<p>Get the system requirements from <a href="https://www.ibm.com/support/knowledgecenter/SSBS6K_3.1.0/supported_system_config/hardware_reqs.html" rel="nofollow">IBM Knowledge Center</a> for IBM Cloud Private.</p>
</li>
<li>
<p>Setup NFS Server and export one or more path for persistent volume.</p>
</li>
</ul>
<h3><a id="user-content-installing-ibm-cloud-private" class="anchor" aria-hidden="true" href="#installing-ibm-cloud-private"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installing IBM Cloud Private</h3>
<p>Following <a href="https://www.ibm.com/support/knowledgecenter/SSBS6K_3.1.0/installing/install.html" rel="nofollow">installation steps in IBM Knowledge Center</a> to install IBM Cloud Private 3.1.0 or later with master, proxy, worker, and optional management and vulnerability advisory nodes in your cluster in standard or high availability configurations.</p>
<p>The guide takes IBM Cloud Private 3.1.0 as example below. You can check the IBM Cloud Private after installation.</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> kubectl get node</span>
NAME         STATUS    ROLES                      AGE       VERSION
10.43.0.38   Ready     management                 11d       v1.11.1+icp-ee
10.43.0.39   Ready     master,etcd,proxy          11d       v1.11.1+icp-ee
10.43.0.40   Ready     va                         11d       v1.11.1+icp-ee
10.43.0.44   Ready     worker                     11d       v1.11.1+icp-ee
10.43.0.46   Ready     worker                     11d       v1.11.1+icp-ee
10.43.0.49   Ready     worker                     11d       v1.11.1+icp-ee</pre></div>
<h3><a id="user-content-creating-image-policy-and-persistent-volume" class="anchor" aria-hidden="true" href="#creating-image-policy-and-persistent-volume"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Creating image policy and persistent volume</h3>
<ol>
<li>
<p>Create Kubernetes namespace.</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-k">export</span> K8S_NAMESPACE=kubeflow
kubectl create namespace <span class="pl-smi">$K8S_NAMESPACE</span></pre></div>
<ul>
<li><strong>K8S_NAMESPACE</strong> is namespace name that the Kubeflow will be installed in. By default should be "kubeflow".</li>
</ul>
</li>
<li>
<p>Create image policy for the namespace.</p>
<p>The image policy definition file (<code>image-policy.yaml</code>) is as following:</p>
<div class="highlight highlight-source-yaml"><pre><span class="pl-ent">apiVersion</span>: <span class="pl-s">securityenforcement.admission.cloud.ibm.com/v1beta1</span>
<span class="pl-ent">kind</span>: <span class="pl-s">ImagePolicy</span>
<span class="pl-ent">metadata</span>:
  <span class="pl-ent">name</span>: <span class="pl-s">image-policy</span>
<span class="pl-ent">spec</span>:
  <span class="pl-ent">repositories</span>:
    - <span class="pl-ent">name</span>: <span class="pl-s">docker.io/*</span>
      <span class="pl-ent">policy</span>: <span class="pl-c1">null</span>
    - <span class="pl-ent">name</span>: <span class="pl-s">k8s.gcr.io/*</span>
      <span class="pl-ent">policy</span>: <span class="pl-c1">null</span>
    - <span class="pl-ent">name</span>: <span class="pl-s">gcr.io/*</span>
      <span class="pl-ent">policy</span>: <span class="pl-c1">null</span>
    - <span class="pl-ent">name</span>: <span class="pl-s">ibmcom/*</span>
      <span class="pl-ent">policy</span>: <span class="pl-c1">null</span>
    - <span class="pl-ent">name</span>: <span class="pl-s">quay.io/*</span>
      <span class="pl-ent">policy</span>: <span class="pl-c1">null</span></pre></div>
<p>Create ImagePolicy for the specified namespace.</p>
<div class="highlight highlight-source-shell"><pre>kubectl create -n <span class="pl-smi">$K8S_NAMESPACE</span> -f image-policy.yaml </pre></div>
</li>
<li>
<p>Create persistent volume (PV) for Kubeflow components.</p>
<p>Some Kubeflow components need PVs to storage data, such as minio, mysql katib. We need to create PVs for those pods in advance.
The PVs defination file (<code>pv.yaml</code>) is as following:</p>
<div class="highlight highlight-source-yaml"><pre> <span class="pl-ent">apiVersion</span>: <span class="pl-c1">v1</span>

<p>Follow these steps to deploy Kubeflow:</p>
<ol>
<li>
<p>Download a <code>kfctl</code> v0.5.0 or later release from the <a href="https://github.com/kubeflow/kubeflow/releases/">Kubeflow releases page</a>.</p>
</li>
<li>
<p>Unpack the tar ball:</p>
<div class="highlight highlight-source-shell"><pre>tar -xvf kfctl_<span class="pl-k">&lt;</span>release tag<span class="pl-k">&gt;</span>_<span class="pl-k">&lt;</span>platform<span class="pl-k">&gt;</span>.tar.gz</pre></div>
</li>
<li>
<p>Run the following commands to set up and deploy Kubeflow. The code below
includes an optional command to add the binary <code>kfctl</code> to your path. If you don't add the binary to your path, you must use the full path to the <code>kfctl</code>
binary each time you run it.</p>
<div class="highlight highlight-source-shell"><pre><span class="pl-c"><span class="pl-c">#</span> The following command is optional, to make kfctl binary easier to use.</span>
<span class="pl-k">export</span> PATH=<span class="pl-smi">$PATH</span>:<span class="pl-k">&lt;</span>path to kfctl <span class="pl-k">in</span> your Kubeflow installation<span class="pl-k">&gt;</span>
<span class="pl-k">export</span> KFAPP=<span class="pl-k">&lt;</span>your choice of application directory name<span class="pl-k">&gt;</span>
<span class="pl-c"><span class="pl-c">#</span> Installs Istio by default. Comment out Istio components in the config file to skip Istio installation.</span>
<span class="pl-k">export</span> CONFIG=<span class="pl-s"><span class="pl-pds">"</span>https://raw.githubusercontent.com/kubeflow/kubeflow/v0.6-branch/bootstrap/config/kfctl_k8s_istio.0.6.2.yaml<span class="pl-pds">"</span></span>

kfctl init <span class="pl-smi">${KFAPP}</span> --config=<span class="pl-smi">${CONFIG}</span> -V
<span class="pl-c1">cd</span> <span class="pl-smi">${KFAPP}</span>
kfctl generate all -V
kfctl apply all -V</pre></div>
<ul>
<li><strong>${KFAPP}</strong> - the <em>name</em> of a directory where you want Kubeflow
configurations to be stored. This directory is created when you run
<code>kfctl init</code>. If you want a custom deployment name, specify that name here.
The value of this variable becomes the name of your deployment.
The value of this variable cannot be greater than 25 characters. It must
contain just the directory name, not the full path to the directory.
The content of this directory is described in the next section.</li>
</ul>
</li>
<li>
<p>Check the resources deployed in namespace <code>kubeflow</code>:</p>
<div class="highlight highlight-source-shell"><pre>kubectl -n kubeflow get  all</pre></div>
</li>
</ol>
<h2><a id="user-content-access-kubeflow-dashboard" class="anchor" aria-hidden="true" href="#access-kubeflow-dashboard"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Access Kubeflow dashboard</h2>
<p>From Kubeflow 0.6, the Kubeflow Dashboard can be accessed via <code>istio-ingressgateway</code> service. If loadbalancer is not available in your environment, NodePort or Port forwarding can be used to access the Kubeflow Dashboard. Refer <a href="https://istio.io/docs/tasks/traffic-management/ingress/ingress-control/" rel="nofollow">Ingress Gateway guide</a>.</p>
<p>For Kubeflow version lower than 0.6, access the Kubeflow Dashboard via Ambassador service. Change the Ambassador service type to NodePort, then access the Kubeflow dashboard through Ambassador.</p>
<div class="highlight highlight-source-shell"><pre>kubectl -n kubeflow patch service ambassador -p <span class="pl-s"><span class="pl-pds">'</span>{"spec":{"type": "NodePort"}}<span class="pl-pds">'</span></span>

<pre><code>http://${MANAGEMENT_IP}:$AMBASSADOR_PORT/

<h3><a id="user-content-insufficient-pods" class="anchor" aria-hidden="true" href="#insufficient-pods"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Insufficient Pods</h3>
<p>Default installation of IBM Cloud Private configures 10 allocatable pods per core. So if you have 8 cores, there will be 80 allocatable pods. However, execution of IBM Cloud Private (from default installation) requires ~40 pods and Kubeflow requires ~45 pods, so you may have insufficient pods issue.</p>
<p>To increase the number of pods, you can edit /etc/cfc/kubelet/kubelet-service-config, increase podsPerCore from 10 to 20 (or 30). Save your change and restart kubelet.</p>
<pre><code>systemctl restart kubelet
</code></pre>
